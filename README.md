# 文言雀 —— 三端功能规划

> 本文档为产品功能规划，记录三端（用户端、家长端、教师端）的完整功能蓝图。待实际开发时再细化技术方案。

---

## 整体架构

### 端形态

三个独立微信小程序，各自开发、独立审核：

| 端 | 小程序名称 | 用户 | 定位 |
|------|-----------|------|------|
| 用户端 | 文言雀（现有） | 学生 | 每日学习 + 答题 + 排行榜 + 查看奖励进度 |
| 家长端 | 文言雀·家长 | 家长 | 绑定孩子 + 学情查看 + 考察验收 + 勋章下单 + 自定义奖励 |
| 教师端 | 文言雀·教师 | 语文教师 | 校班登记 + 学情看板 + 任务布置 + 班级打卡奖励 |

### 公共数据层：标准学校班级库

学校和班级由教师端提交、平台后台审核后纳入标准数据库，作为三端的公共基础设施。

```
                平台标准校班库（审核制）
               /         |          \
         用户端        家长端        教师端
    (选学校班级加入)  (选班级找孩子)  (登记+管理)
```

- 同一班级可关联多位教师（如语文老师、班主任）
- 同一班级的学生数据天然聚合，无需教师手动建群拉人
- 排行榜基于标准学校班级库的 region/grade 字段聚合

### 亲子绑定

学生在用户端生成邀请码（或二维码），家长在家长端输入绑码建立关联。一个家长可绑定多个孩子。

## 一、用户端（文言雀 · 现有 + 新增）

### 现有功能（已完成）

- 8 本词书 + 两种学习模式（standard / identify_first）
- 核心学习回路（答题 → 纠错 → 字总结 → 完成）
- 艾宾浩斯调度 + 错题本 + 生词本
- 勋章墙（系统自动发放 8 枚累计天数勋章）
- 打卡日历 + 全局搜索 + 语音播报 + 答题音效
- 选篇板块（77 篇名篇 + 3 种阅读模式 + 典故注释）
- 经典板块（7 部已上线，45 部待开发）

### 新增功能

#### 1. 加入班级

- 入口：「我的」→「加入班级」
- 方式：输入班级邀请码，或扫描教师分享的小程序码
- 填写真实姓名（供教师在花名册中识别）
- 绑定后，教师端花名册实时更新

#### 2. 排行榜

入口：学习首页排行榜区域，三个 Tab 切换。

| 榜单 | 范围 | 聚合维度 | 姓名展示 |
|------|------|---------|:--:|
| 班级榜 | 本班内部 | 累计掌握词数 / 连续打卡天数 / 本周正确率 / 本周学习时长 | 真实姓名 |
| 年级榜 | 全市同年级（跨校） | 累计掌握词数 / 连续打卡天数 / 本月正确率 | 脱敏（张*三 或 XX中学 张同学） |
| 同城榜 | 全市全学段 | 累计掌握词数 / 连续打卡天数 / 勋章数量 | 脱敏 |

关键设计：
- 排名每日凌晨定时计算，非实时
- 用户可在设置中选择是否参与年级榜/同城榜
- 无学校绑定的学生只参与纯人数统计，不参与地理榜
- 后端需对异常答题行为做风控，防止刷榜

#### 3. 自定义奖励进度展示

- 学习首页/词书详情页展示进度条 + 奖品预览
- 家长奖励："🎁 距乐高一套还差 32 词"
- 教师奖励："🔥 连续打卡 18/21 天，距免作业还差 3 天"
- 全班进度（教师可选是否开放）："全班 28/35 人已达标"
- 达成时弹窗祝贺，显示"等待家长/教师确认"

#### 4. 绑定家长

- 入口：「我的」→「绑定家长」
- 生成邀请码/二维码，发送给家长绑定
- 管理已绑定的家长列表，可解除绑定

#### 5. 班级打卡进度

- 学习首页展示本班今日打卡情况
- 班级打卡奖励进度追踪

## 二、家长端（文言雀·家长）

### 功能地图

#### 1. 亲子绑定（P0）

- 输入学生在用户端生成的邀请码 / 扫描二维码
- 支持绑定多个孩子，切换查看
- 也可通过"学校 → 年级 → 班级 → 学生姓名"查找绑定

#### 2. 学情查看（P0）

| 功能 | 说明 |
|------|------|
| 学习概览 | 今日学习时长、答题数、正确率、连续打卡天数 |
| 进度追踪 | 各本词书完成进度、已掌握/模糊/困难词汇分布 |
| 错题查看 | 孩子的错题本，按字展开查看具体错误句子 |
| 勋章墙 | 孩子已获得的系统勋章和考察勋章 |
| 榜单查看 | 孩子在班级榜/年级榜/同城榜的排名 |

#### 3. 家长考察（P1）

定位：面对面口头考察，从"再认"升级到"回忆"，家长人工判断对错。

```
家长选考察范围（某本词书 / 已掌握词 / 错题本）
    ↓
大字显示单个文言字（无拼音、无释义提示）
    ↓
孩子口头说出释义
    ↓
家长点击「显示答案」——展开标准释义
    ↓
家长判断：✅ 正确 / ❌ 错误
    ↓
进入下一个字，直到考察结束
    ↓
结果汇总：正确率 + 每个字的掌握情况
```

关键设计：
- **答案默认隐藏**：家长点击后才展示，避免无意提示孩子
- **考察记录独立存储**：不计入艾宾浩斯体系，作为独立的"家长验收记录"
- **离线可用**：亲子面对面场景可能无网络，考察功能支持离线操作，联网后同步

#### 4. 勋章体系

三种勋章类型：

| 类型 | 触发方式 | 实物支持 | 谁兑现 |
|------|---------|:--:|------|
| 系统勋章 | 连续打卡天数达标，系统自动发 | — | 平台 |
| 考察勋章 | 家长考察通过（如某本词书全部验收） | ✅ 可邮寄实物 | 平台 |
| 自定义奖励 | 家长设定目标+奖品描述 | ❌ | 家长线下自购 |

**考察勋章**：某本词书全部通过家长考察 → 解锁对应考察勋章。与系统自动发放的累计天数勋章区分，含金量更高。

**实物勋章邮寄流程（P2）**：

```
孩子获得考察勋章
  → 家长端该勋章卡片出现「邮寄实物」按钮
  → 填写收货地址
  → 微信支付
  → 平台制作发货
  → 订单状态追踪
```

#### 5. 自定义奖励（P1）

定位：家长和孩子之间的私下约定，平台只做进度追踪，家长自己兑现。

| 步骤 | 家长端 | 用户端 |
|------|--------|--------|
| 设定目标 | 选择触发条件（词书进度 / 连续打卡天数）+ 奖品描述（文字 + 可选图片） | — |
| 追踪中 | 查看进行中的约定列表 | 学习首页显示进度条 + 奖品预览 |
| 达成 | 收到达成通知 → 点击「已发放」确认兑现 | 弹窗祝贺，显示"等待家长确认" |

## 三、教师端（文言雀·教师）

### 功能地图

#### 1. 学校班级登记（P0）

由教师端主动发起，平台后台审核后纳入标准学校班级库。

流程：
```
教师提交 → 平台审核 → 纳入标准库 → 全网可用
```

提交信息：
- 学校名称 + 所在地区（省/市/区）+ 学段（初中/高中/完中）
- 年级 + 班级名称（如"初三3班"）+ 学期（如 2026 秋季）
- 教师身份证明材料（教师资格证/工作证照片）

审核逻辑：
- 查重：该学校是否已在标准库中？
  - 存在 → 该班级是否已存在？→ 存在则提示"该班级已有教师管理，是否申请加入？"
  - 不存在 → 创建学校 → 创建班级 → 审核通过
- 身份验证：核对教师资格证明材料

#### 2. 班级管理（P0）

| 功能 | 说明 |
|------|------|
| 班级邀请码 | 审核通过后自动生成，教师分享到班级微信群或打印张贴 |
| 学生花名册 | 已加入学生列表，含真实姓名、最近学习时间、掌握词数 |
| 移除学生 | 解除学生与班级的关联 |
| 加入提醒 | 查看未加入学生名单（教师可手动登记预期学生姓名做对比） |
| 多班级 | 一个教师可管理多个班级 |
| 同班教师 | 同一班级可关联多位教师（如语文老师+班主任），各自独立 |

#### 3. 学情看板（P0）

| 功能 | 说明 |
|------|------|
| 班级概览 | 总人数、活跃人数、平均正确率、平均学习时长、连续打卡均值 |
| 词书进度 | 指定词书的班级完成率分布（未开始/进行中/已完成） |
| 高频错词 | 班级内错误率最高的字词 TOP 20，支持展开查看具体错误句子 |
| 趋势图表 | 班级周均正确率变化、日均学习时长趋势 |
| 学段筛选 | 按中考/高考维度筛选数据，对齐教学进度 |

#### 4. 个体追踪（P1）

| 功能 | 说明 |
|------|------|
| 学生详情 | 从花名册点击任一学生，查看完整学习数据 |
| 个人进度 | 各本词书完成度、每个词掌握程度（困难/模糊/熟悉/掌握） |
| 个人错题 | 该生错题本查看 |
| 家长考察记录 | 家长验收结果（如有绑定） |
| 学习活跃 | 最近学习时间、近 7 天/30 天学习时长趋势 |
| 导出报表 | 单个学生学习报告（图片/PDF），可用于家长会 |

#### 5. 任务布置（P1）

| 功能 | 说明 |
|------|------|
| 发布任务 | 指定词书 + 截止日期 + 任务描述（如"本周完成中考实词虚词 20 个新词"） |
| 任务类型 | 学新词 / 复习 / 不限 |
| 学生端展示 | 用户端首页显示任务卡片 + 进度 |
| 完成统计 | 教师端查看：已完成/未完成/进行中人数及名单 |
| 催交提醒 | 对未完成学生发送微信订阅消息提醒 |

#### 6. 班级打卡管理 + 奖励（P1）

打卡看板：
| 功能 | 说明 |
|------|------|
| 今日打卡 | 已打卡 X 人 / 未打卡 Y 人，具体名单 |
| 打卡排名 | 班级内连续打卡天数排行、累计打卡天数排行 |
| 异常提醒 | 连续 N 天未打卡学生标红 |
| 历史趋势 | 班级每日打卡人数趋势图（本月/本学期） |

班级打卡奖励（与家长自定义奖励同一模式）：

```
教师端设定：
  目标（连续打卡 X 天 / 累计打卡 X 天 / 周一到周五全勤）
  + 奖励描述（如"免一次作业""全班看电影"）
    ↓
用户端展示：
  学习首页显示班级打卡进度 + 奖励预览
  → 个人："🔥 连续打卡 18/21 天，距免作业还差 3 天"
  → 全班（可选）"全班 28/35 人已达标"
    ↓
达标后：
  教师端收到通知 → 确认「已兑现」
  用户端弹窗祝贺
```

#### 7. 内容反馈管理（P2）

| 功能 | 说明 |
|------|------|
| 反馈列表 | 学生提交的词书内容错误反馈汇总 |
| 反馈处理 | 标记"已确认"/"已修复"/"无需修改"，附处理备注 |
| 反馈统计 | 哪些词书/字的反馈最多，辅助内容优化 |
| 教师纠错 | 教师也可主动提交内容纠错 |

#### 8. 可能的扩展方向（P3）

| 功能 | 说明 |
|------|------|
| 校本词书 | 教师自定义词书（上传校本词汇表），本班可见 |
| 教学资源推送 | 推送选篇/经典阅读材料给学生作为课外阅读 |
| 教研协作 | 同校教师共享词书/考察计划 |

## 四、优先级路线图

### 第一阶段：基础绑定 + 家长学情查看
- 用户端：加入班级、绑定家长（生成邀请码）
- 家长端：亲子绑定、学情查看（概览+进度+错题+勋章）
- 教师端：学校班级登记（审核制）、班级管理、学生花名册

### 第二阶段：考察 + 排行榜
- 用户端：排行榜（班级榜/年级榜/同城榜）
- 家长端：家长考察、考察勋章、自定义奖励
- 教师端：学情看板、个体追踪

### 第三阶段：教学闭环
- 用户端：任务卡片、班级打卡进度
- 教师端：任务布置+统计、班级打卡管理+奖励
- 家长端：榜单查看

### 第四阶段：体验升级
- 家长端：学习报告（周报/月报）、实物勋章邮寄
- 教师端：内容反馈管理、导出报表、校本词书

## 五、关键设计决策（已确定）

| 决策 | 结论 |
|------|------|
| 端形态 | 三个独立小程序 |
| 学校班级数据 | 教师提交 → 平台审核 → 标准库 → 全网可用 |
| 亲子绑定 | 学生用户端生成邀请码，家长输入绑定 |
| 师生绑定 | 教师端生成班级邀请码，学生输入加入 |
| 教师身份 | 审核制（上传资格证，后台审核） |
| 考察判断 | 家长人工判断对错（非四选一自动判断） |
| 排行榜 | 班级内实名、班级外脱敏 |
| 奖励兑现 | 平台负责勋章实物，家长/教师各自兑现自定义奖励 |
| 排名计算 | 每日凌晨定时跑批，非实时 |

## 六、增长分析

### 增长飞轮

```
教师建班 → 全班学生加入 → 学生邀请家长绑定 → 家长口碑/教师推荐 → 更多教师建班
```

三端之间的绑定关系天然形成 B2B2C 裂变：**教师端是杠杆支点**，撬动一个班的学生，学生再撬动家长。

### 增长链路

```
教师选用 → 全班加入 → 同年级备课组跟进 → 全校语文组采用 → 教研员背书 → 同城跨校推广
```

| 节点 | 传导逻辑 | 可能性 |
|------|---------|:--:|
| 教师→全班 | 教师直接管理，邀请码发群，学生必须加入 | 极高 |
| 全班→全年级 | 同年级备课组统一教研，教学进度/工具选用同步 | 高 |
| 全年级→全校 | 教研组长/学科带头人拍板，中考高考文言文是全校性痛点 | 中高 |
| 全校→同城 | 教研员跨校教研活动推广，同城排行榜驱动校际竞争 | 中 |

关键：**学校一旦采用，基本不会换**。教师用熟了、学生习惯了、数据积累了——每多一所学校，是每年都新增一批用户。

### 动态增长模型

中国每年约 1,500 万新生进入初中，加上高中在校周期共 6 年：

| 维度 | 数据 |
|------|------|
| 每年新增可触达学生 | ~1,500 万 |
| 6 年在校存量总池 | ~9,000 万（滚动更新） |
| 渗透率 10% 对应存量 | ~900 万学生 |
| 渗透率 10% 对应年新增 | ~150 万/年 |
| 10 年累计触达（含毕业生） | 远超千万 |
| 家长端放大系数 | ×1.5（每学生绑定 1-2 位家长） |

增长曲线非线性的核心原因：**存量学校留存 + 新增学校扩张**叠加，形成代际渗透效应。

## 七、盈利模式

### 核心原则

> **学生端核心学习体验永远免费。** 答题、复习、艾宾浩斯、错题本、排行榜——这些学生端功能不设任何付费点。盈利切的是周边服务和成人用户（家长、教师、学校）。

| 永远免费（学生端） | 可以收费（周边 + 成人） |
|----------|----------|
| 答题、复习、艾宾浩斯 | 实物勋章邮寄 |
| 错题本、生词本 | 家长学情报告导出 |
| 打卡日历 | 教师高级看板 |
| 排行榜 | 学校/教育局采购 |
| 词书、选篇、经典阅读 | 知识付费精品课 |
| 语音播报、音效 | 校本词书定制 |

### 盈利切入点

按对用户体验的伤害程度从低到高排列：

#### 第一层：零伤害（学生完全感知不到）

**B2B 学校/区域采购**
- 教育局为全区购买教师端高级功能，按区域年费 5-20 万
- 学校采购高级学情看板、批量导出，500-2000 元/年/校
- 覆盖 1000 所学校 → 年收入 200-500 万

**出版社合作**
- 与教材/教辅出版社的内容授权或联合推广
- 平台积累的高频错词、学情数据（脱敏）授权出版社优化教辅内容

#### 第二层：低频感知（用户偶尔看到）

**实物勋章商城**
- 孩子通过家长考察 → 家长付费购买实物勋章（29-39 元/枚）
- 节日限定、年度限定勋章，收藏属性驱动复购
- 千万学生中 2% 购买 → 年收入约 800 万
