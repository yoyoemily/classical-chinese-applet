## 壳文章 (`art_shell_001` ~ `art_shell_121`) 的定位

### 是什么

壳文章是**词书 keyWords 的句子上下文容器**，不是用户可读的选篇。在 v2 架构中，词书通过 `keyWordRefs[].kid` 引用选篇 keyWords，壳文章的句子和 keyWords 就是这些引用的落地锚点——词书需要真实的句子环境来承载生词，但文章本身不需要对用户展示。

### 系统行为

| 维度 | 行为 |
|------|------|
| 文章列表 | **不可见**。后端 `ArticleService.getArticles()` 强制 `.eq(Article::getHasContent, 1)`，壳文章全部 `has_content = 0`，被过滤 |
| 文章详情 | API 无过滤，但用户正常路径走不到；直接输 URL 可以打开阅读器 |
| 排序 | `sortOrder = 10000 + index`，排在大纲文章之后 |
| 数据源 | `articles.json`（知识库），共 121 篇、339 句、339 条 keyWords |

### 为什么不删

不是所有词书的 keyWord 都能落在大纲文章里。大纲只有 77 篇部编版教材篇目，覆盖面有限。壳文章补充了更多句子场景，让词书的每个 keyWordRef 都有一个句子可依附。

### 译文要求

- 壳文章本身不需要译文（`has_content=0`，用户不可见）
- **但被词书引用到的句子必须有译文**，因为学生在词书答题时会看到该句作为上下文，缺译文导致学习回路不完整
- 判断标准：有 keyWord 的句子 → 必须有译文；无 keyWord 的句子 → 译文可选
- 当前状态（2026-07-20）：48 句有 keyWord 但缺译文，分布在 18 篇壳文章中，需补齐

### 数据完整性标准对比

| | 大纲文章 (77篇) | 壳文章 (121篇) |
|----|----|----|
| 用户可见 | ✅ 列表可浏览 | ❌ 仅词书引用时间接出现 |
| keyWords | 完整标注（1,095 条） | 按词书需求标注（339 条） |
| 典故注释 | 77 篇全部完成 | 无 |
| 创作背景 | 77 篇全部完成 | 无 |
| 译文 | **100%**（逐句） | **有 keyWord 的句子必须 100%**，其余不强制 |
| 语音播报 | 部分有 | 无 |
